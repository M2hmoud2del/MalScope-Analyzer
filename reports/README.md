# 📄 PDF Report Generator

## What This Module Does

The PDF Report Generator takes the finalized analytical data (Static, Dynamic, and AI Insights) and compiles it into a **professional, SOC-ready PDF document**. 

Instead of requiring analysts to manually screenshot the GUI or parse raw JSON, it:
1. **Extracts** the core details (File Name, Hash, Score, Engine Verdict).
2. **Injects** the AI-generated Threat Intelligence Summary.
3. **Formats** technical indicators (Static properties & Dynamic behaviors) into clean, readable lists.
4. **Auto-saves** the document into a dedicated `generated_reports/` directory with a timestamped filename to maintain an audit trail.

## Technical Concepts Applied

| Capability | Implementation Detail |
| :--- | :--- |
| **Automated Reporting Pipeline** | Transforming in-memory JSON data structures into persistent, standardized document formats. |
| **Dynamic Document Generation** | Using programmatic canvas drawing (`fpdf2`) to handle variable-length text and dynamic data arrays without breaking page layouts. |
| **Data Sanitization** | Safely parsing nested dictionaries (using `.get()` methods and type checking) to prevent runtime crashes if a module returns incomplete data. |

## Input Format

The module expects the raw file dictionary (from the Orchestrator) and the AI explanation string.

```python
target_data = {
    "file": "sample.exe",
    "verdict": "malicious",
    "score": 95,
    "static": {"entropy": 6.8, "urls": ["[http://malicious.com](http://malicious.com)"]},
    "dynamic": {"processes": ["cmd.exe"]}
}

ai_text = "1. Threat Summary: This file is a confirmed dropper..."

```

## Example Output

The function returns the absolute string path to the newly created PDF file.

```text
"/Users/username/MalScope-Analyzer/generated_reports/MalScope_Report_sample_exe.pdf"

```

**Inside the PDF, the layout includes:**

* **Executive Summary:** File name, overall risk score, and final verdict.
* **AI Threat Intelligence:** The step-by-step reasoning and recommendations generated by the LLM.
* **Technical Telemetry:** Clean lists of spawned processes, network connections, and static anomalies.

## How to Test Independently

To test the PDF generator without running the full GUI or triggering an actual scan:

```bash
# Run the generator directly from the root directory
python -m reports.pdf_generator

```

*(Note: Ensure you have run `pip install fpdf2` first).*

## Integration with Orchestrator

The orchestrator calls this module when the `request_report` signal is triggered by the user clicking the "Generate PDF" button in the GUI:

```python
from reports.pdf_generator import ReportGenerator

# Reconstruct data safely
report_gen = ReportGenerator()
pdf_path = report_gen.generate_pdf(target_data, ai_text)

```

The resulting `pdf_path` is emitted to the UI's Log Console to confirm successful creation and notify the user where the file was saved.
